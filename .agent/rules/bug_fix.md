# Beeliber Web - Bug Fix Skill

이 가이드는 Beeliber 웹 애플리케이션의 버그 수정 요청 발생 시 준수해야 할 표준 절차와 원칙을 정의합니다.

## 버그 수정 원칙

1.  **원인 특정 우선**: 수정을 시작하기 전에 반드시 버그의 근본 원인을 정확히 특정합니다.
2.  **최소 범위 수정**: 원인 해결에 필요한 최소한의 코드만 수정하여 사이드 이펙트를 방지합니다.
3.  **근본적 해결**: 단순히 증상만 가리는 임시 방편이 아닌, 동일한 문제가 재발하지 않도록 견고하게 처리합니다.
4.  **불필요한 변경 금지**: 요청되지 않은 리팩토링이나 스타일 변경 등 작업 범위 밖의 수정은 하지 않습니다.
5.  **영향 범위 보고**: 공통 로직이나 전역 상태를 수정해야 할 경우, 그로 인한 파급 효과를 명확히 분석하여 보고합니다.
6.  **검증 프로세스**: 수정 완료 후 기존의 정상적인 비즈니스 플로우가 유지되는지 확인할 수 있는 검증 포인트를 제시합니다.

## 분석 및 보고 형식 (필수 포함 내용)

버그 수정 완료 후 다음과 같은 형식으로 보고합니다.

1.  **버그 증상**: 발생한 문제에 대한 구체적인 설명
2.  **원인 추정/확인**: 코드 분석 또는 로그를 통해 발견한 버그의 근본 원인
3.  **수정 범위**: 변경이 적용된 파일 및 코드 블록
4.  **수정 방식**: 어떤 방식으로 문제를 해결했는지 기술
5.  **영향 가능 영역**: 수정으로 인해 확인이 필요한 주변 기능
6.  **테스트 시나리오**: 정상 동작을 확인하기 위해 수행해야 할 단계별 시나리오

## 금지 사항

*   버그와 관련 없는 코드 정리 (Lint 수정, 주석 정리 등 일괄 적용 금지)
*   스타일 변경을 동반한 불필요한 레이아웃 수정
*   파일의 대규모 재구성 및 폴더 구조 변경
