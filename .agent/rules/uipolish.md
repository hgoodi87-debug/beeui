# 🎨 UI Polish (UI 다듬기 모드) 규칙

## 언제 활성화하나
- 시각적 완성도 개선 요청 시
- 간격, 폰트, 컬러, 정렬 조정 시
- "예쁘게 해줘", "더 프리미엄하게", 스샷 기반 수정 요청 시

## 반드시 참조할 스킬

`beeliber_design` — 아래 항목 필수 준수:
- 컬러: `#FFC700` (Yellow) / `#000000` (Black) / `#FFFFFF` (White)
- 폰트: Pretendard 또는 Inter, `letter-spacing: -0.05em`
- 분위기: Warm & Bright (어둡고 무거운 필터 금지)
- 로고 `liber` 부분: 15도 이탤릭 유지

## UI 수정 원칙

1. **로직 보존** — API, 상태 관리, 이벤트 핸들러 절대 수정 금지
2. **영향 범위 최소화** — 공통 컴포넌트보다 로컬 컴포넌트에서 해결
3. **반응형 유지** — 기존 반응형 레이아웃 동작 유지
4. **텍스트 유지** — 문구 변경 요청 없으면 기존 텍스트 그대로

## 수정 보고 형식

→ 상세 절차는 `.agent/rules/ui_patch.md` 참조

```
UI 수정 요약 → 변경된 화면 영역 → 변경 파일 → 유지된 로직 → QA 포인트
```
