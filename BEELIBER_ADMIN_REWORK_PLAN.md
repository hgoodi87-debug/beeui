# Beeliber 본사 어드민 통합 개편 기획안

## 문서 목적

본 문서는 Beeliber 본사 어드민의 핵심 운영 화면인 아래 3개 영역을 하나의 흐름으로 재설계하기 위한 통합 개편안이다.

- 통합 관제 대시보드
- 예약 관리 페이지
- 일일 정산 페이지

이번 개편의 목표는 화면을 단순히 보기 좋게 만드는 것이 아니라, 본사 운영자가 아래 흐름을 더 빠르고 정확하게 수행하도록 만드는 것이다.

**상황 파악 → 예약 처리 → 정산 마감**

---

## 핵심 문제 정의

현재 어드민은 각 페이지가 개별적으로는 동작하지만, 운영 흐름상 아래 문제가 있다.

1. 대시보드는 숫자를 보여주지만 지금 당장 무엇을 처리해야 하는지 약하다.
2. 예약 관리는 목록 조회 중심이며 실무 처리 우선순위가 약하다.
3. 일일 정산은 매출/예약/시재 정보가 섞여 있어 마감 화면으로서의 통제력이 약하다.
4. 세 화면이 연결되지 않아, 운영자가 페이지를 옮겨 다니며 수동으로 판단해야 한다.
5. 운영 상태, 결제 상태, 정산 상태가 서로 분리되어 보이지 않아 예외 처리 누락 위험이 있다.

---

## 개편 목표

이번 개편은 아래 6가지를 달성해야 한다.

1. 오늘 운영 현황을 10초 안에 파악할 수 있어야 한다.
2. 예약을 단순 조회가 아니라 우선순위 기반으로 처리할 수 있어야 한다.
3. 완료된 주문이 정산까지 자연스럽게 이어져야 한다.
4. 지점별 병목, 지연, 차액, 이슈가 바로 드러나야 한다.
5. 정산은 매출 집계가 아니라 마감 운영 시스템처럼 동작해야 한다.
6. 추후 월 정산, 파트너 정산, 운송사 정산으로 확장 가능한 구조여야 한다.

# 1. 전체 UX 구조 재정의

## 역할 분리 원칙

세 화면의 역할을 아래처럼 명확히 분리한다.

### 1) 통합 관제 대시보드

- 목적: 오늘 전체 운영 상황을 빠르게 파악
- 성격: 상황판
- 핵심 질문:
  - 오늘 예약은 얼마나 들어왔는가?
  - 지금 가장 먼저 처리해야 할 것은 무엇인가?
  - 지연/이슈가 있는가?
  - 정산과 연결되지 않은 건이 있는가?

### 2) 예약 관리

- 목적: 실제 예약을 처리하고 상태를 관리
- 성격: 작업판
- 핵심 질문:
  - 지금 처리해야 할 예약은 무엇인가?
  - 어떤 예약이 지연/보류/취소 상태인가?
  - 이 예약은 결제/정산까지 반영되었는가?

### 3) 일일 정산

- 목적: 오늘 돈 흐름을 확정하고 마감
- 성격: 마감판
- 핵심 질문:
  - 오늘 정산 확정 금액은 얼마인가?
  - 미확정/보류 건은 무엇인가?
  - 현금 시재 차액이 있는가?
  - 파트너/운송사 지급 예정액은 얼마인가?

# 2. 통합 관제 대시보드 개편안

## 목표

운영자/대표/본사 관리자가 대시보드에서 오늘 전체 상황과 우선 작업을 빠르게 파악할 수 있도록 한다.

## 현재 문제

- 배송/보관 상태 숫자는 있으나 우선순위가 약함
- 오늘 해야 할 작업이 분리되어 보이지 않음
- 예약 관리 / 정산 페이지로 이어지는 실무 동선이 약함
- 매출 정보가 운영 정보보다 먼저 강조됨

## 개편 방향

숫자 중심 대시보드에서 **행동 유도형 대시보드**로 전환한다.

## 추천 섹션 구조

### A. 상단 KPI 영역

기존 배송/보관 대기 카드 대신 아래 KPI를 상단 1열 우선 노출한다.

- 오늘 총 예약
- 오늘 신규 결제
- 오늘 입고 예정
- 오늘 출고/공항 연계 예정
- 지연/이슈 건수
- 오늘 정산 확정액

설계 원칙:

- 운영 숫자가 매출 숫자보다 우선
- 각 KPI 클릭 시 해당 필터가 적용된 예약 관리 또는 정산 화면으로 이동

### B. 오늘 해야 할 작업

신규 블록 추가 필수.

권장 카드:

- 입고 확인 필요
- 출고/연계 처리 필요
- 취소/변경 요청 대기
- 정산 미확정 건

각 카드에는 아래를 포함한다.

- 건수
- 중요도 색상
- 바로가기 버튼

### C. 운영 상태 보드

보관/배송 현황 블록을 아래 형태로 재구성한다.

#### Delivery

- 오늘 접수 예정
- 이동중
- 도착 대기
- 지연
- 이슈

#### Storage

- 오늘 입고 예정
- 보관중
- 오늘 픽업 예정
- 미입고
- 초과보관

### D. 지점별 현황

서울 지점별 운영 상태를 한눈에 보는 블록 추가.

컬럼 예시:

- 지점명
- 오늘 예약
- 입고 완료
- 보관중
- 출고 예정
- 이슈 건수
- 현금 마감 상태
- 최종 업데이트 시간

### E. 매출/정산 요약

현재 오늘 매출, 주간 추이 영역은 유지하되 하단으로 내린다.

권장 지표:

- 오늘 총 결제액
- 오늘 환불액
- 오늘 정산 확정액
- 주간 매출 추이
- 정산 보류 추이

## 대시보드 액션 흐름

- KPI 클릭 → 예약 관리 또는 정산 필터 적용 화면 이동
- 이슈 카드 클릭 → 이슈/보류 필터 자동 적용
- 지점별 현황 클릭 → 해당 지점 중심 예약/정산 보기 이동

# 3. 예약 관리 페이지 개편안

## 목표

예약 목록 조회 화면이 아니라 **실무 처리 중심 화면**으로 바꾼다.

## 현재 문제

- 목록은 깔끔하지만 “지금 처리해야 할 예약” 우선순위가 약함
- 운영 상태, 결제 상태, 정산 상태가 분리되어 보이지 않음
- 상세 처리를 위해 여러 화면 이동이 필요할 가능성이 큼
- 탭 구성이 실제 업무 흐름보다 포괄적임

## 개편 방향

리스트형 조회 화면에서 **작업형 관리 화면**으로 개편한다.

## 추천 구조

### A. 상단 상태 요약 바

상단에 상태 요약 KPI를 추가하여 필터 진입을 빠르게 만든다.

권장 항목:

- 오늘 전체 예약
- 입고 대기
- 보관중
- 오늘 픽업 예정
- 취소/환불
- 이슈/보류
- 미결제
- 정산 미반영

각 항목 클릭 시 테이블 필터 자동 적용.

### B. 탭 구조 재설계

현재 `전체 / 대기중 / 진행중 / 완료됨`을 더 실무적으로 쪼갠다.

#### 보관 예약 관리 추천 탭

- 전체
- 오늘 입고 예정
- 보관중
- 오늘 픽업 예정
- 취소/환불
- 이슈/보류
- 완료

#### 배송 예약 관리 추천 탭

- 전체
- 오늘 접수 예정
- 이동중
- 도착 대기
- 완료
- 이슈/보류

### C. 필터 영역 강화

필수 필터:

- 날짜
- 지점
- 서비스 유형
- 운영 상태
- 결제 상태
- 정산 상태
- 이슈 여부
- 공항 연계 여부
- 외국어 고객 여부

### D. 테이블 컬럼 개편

기존 테이블에 아래 정보를 추가/정렬한다.

권장 컬럼:

- 예약번호
- 고객명 / 연락처
- 서비스 유형
- 지점명
- 예약 시간
- 운영 상태
- 결제 상태
- 정산 상태
- 이슈 여부
- 담당 액션

운영 상태 / 결제 상태 / 정산 상태는 각각 분리된 배지로 표현.

예시:

- 운영 상태: 보관중
- 결제 상태: 결제완료
- 정산 상태: 미반영

### E. 행 액션 구조 개편

현재 아이콘형 액션은 유지 가능하나 반드시 라벨 또는 드롭다운 메뉴로 의미를 명확히 한다.

필수 액션:

- 상세보기
- 상태변경
- 고객연락
- 예외처리

### F. 우측 상세 패널 추가

행 클릭 시 우측 슬라이드 상세 패널이 열리도록 설계.

포함 정보:

- 고객 정보
- 짐 정보
- 예약 시간
- 상태 타임라인
- 결제 정보
- 정산 반영 여부
- 메모/이슈
- 사진 증빙

효과:

- 상세 페이지로 이동하지 않고 빠르게 처리 가능
- CS/운영/정산팀의 작업 속도 향상

## 예약 관리와 정산 연결 포인트

예약 관리 화면에 아래 필드를 반드시 추가한다.

- payment_status
- service_completion_status
- settlement_status
- exception_status

예시 settlement_status:

- 미반영
- 정산확정
- 정산보류
- 월정산반영
- 지급완료

# 4. 일일 정산 페이지 개편안

## 목표

매출/현황 화면이 아니라 **실제 일마감을 수행하는 정산 운영 화면**으로 바꾼다.

## 현재 문제

- 예약 목록과 정산 정보가 혼합됨
- 정산 기준값이 약함
- 현금 시재는 있으나 승인/차액 관리 흐름이 약함
- 보류/예외/지급예정 구조가 안 보임

## 개편 방향

**요약 → 마감 상태 → 정산 원장 → 지급 예정 → 예외/조정** 구조로 재편.

## 추천 구조

### A. 상단 정산 KPI

권장 카드:

- 오늘 총 결제액
- 오늘 정산 확정액
- 오늘 미확정액
- 오늘 환불/취소 반영액
- 파트너 지급 예정액
- 운송사 지급 예정액
- 본사 예상 마진
- 정산 보류 건수

### B. 마감 진행 상태

신규 섹션 추가.

예시 단계:

- 예약 발생
- 결제 완료
- 서비스 완료
- 정산 확정
- 일마감 완료
- 지급 대기

표시 정보:

- 총 주문
- 완료 건수
- 정산 확정 건수
- 보류 건수
- 환불 건수
- 차액 확인 필요 건수

### C. 상단 액션 버튼

우측 상단에 아래 액션 묶음 배치.

- 일마감 실행
- 보류건 보기
- 차액 확인
- 리포트 다운로드

### D. 정산 원장 테이블

정산 화면의 핵심 중심 블록.

권장 컬럼:

- 주문번호
- 서비스 유형
- 지점명
- 완료일시
- 고객 결제금액
- 할인/쿠폰
- 환불/취소 반영
- 정산 대상금액
- 파트너 정산금
- 운송사 정산금
- 본사 몫
- 정산 상태
- 지급 상태
- 이슈 여부

필수 필터:

- 날짜
- 지점
- 서비스유형
- 정산상태
- 지급상태
- 이슈여부

### E. 결제수단별 집계 + 현금 시재 마감

현재 우측 영역을 마감 블록으로 강화.

포함 항목:

- 카드
- 현금
- 계좌이체
- 외부결제
- 수기정산

현금 마감 필드:

- 이론상 현금 수납
- 실제 시재 입력
- 차액
- 차액 사유
- 승인 상태
- 승인자

상태값:

- 미마감
- 차액 있음
- 검토 필요
- 마감 완료

### F. 지점별 일일 정산 현황

신규 블록 추가.

권장 컬럼:

- 지점명
- 주문수
- 완료건수
- 정산확정액
- 현금수납
- 차액
- 마감 상태
- 최종 업데이트 시간

### G. 예외 / 조정 관리

신규 블록 추가 필수.

포함 대상:

- 환불 발생 건
- 취소 후 결제 잔존 건
- 서비스 완료 전 결제 건
- 운송 완료인데 정산 누락 건
- 차액 발생 건
- 수동 조정 건

권장 컬럼:

- 주문번호
- 이슈유형
- 금액 영향
- 책임 주체
- 처리 상태
- 담당자
- 처리기한

## 월 정산 확장 전제

일일 정산 데이터에는 아래 필드를 반드시 포함한다.

- settlement_date
- settlement_owner_type
- settlement_owner_id
- settlement_status
- payout_status
- monthly_settlement_batch_id
- adjustment_applied

# 5. 세 화면 연결 구조

## 공통 흐름

대시보드 → 예약 관리 → 일일 정산이 자연스럽게 이어지도록 설계한다.

### 예시 플로우

1. 대시보드에서 `입고 확인 필요 5건` 확인
2. 클릭 시 예약 관리 페이지로 이동, 해당 필터 자동 적용
3. 예약 처리 완료
4. 서비스 완료 건은 정산 대기 상태로 변경
5. 정산 화면에서 해당 건이 정산 원장에 반영
6. 일마감 확정

## 공통 상태 체계

세 화면 모두 아래 상태 체계를 공유한다.

### 운영 상태

- 예약됨

# 6. UI/디자인 공통 가이드

## 색상 규칙

- 정상 완료 / 정산확정: 초록
- 진행중 / 대기: 노랑
- 미결제 / 미확정: 회색
- 이슈 / 지연 / 취소: 빨강
- 보관중: 파랑
- 수동조정 / 관리자개입: 보라

## 배지 규칙

서비스 상태, 결제 상태, 정산 상태를 하나의 배지에 섞지 않는다.
반드시 분리된 3단 구조로 표기한다.

## 인터랙션 원칙

- KPI 카드 클릭 → 필터 적용된 상세 화면 이동
- 테이블 행 클릭 → 우측 상세 패널 오픈
- 고위험 액션(취소, 정산확정, 수동조정)에는 확인 단계 필요

## 반응형 우선순위

- 데스크톱: 다중 카드 + 테이블 + 우측 패널
- 중간 해상도: 요약 카드 + 테이블 중심
- 모바일은 운영용 기준에서 우선순위 낮음, 조회 위주 최소 대응

# 8. 우선순위별 실행안

## P0 (즉시 개편)

- 대시보드에 `오늘 해야 할 작업` 섹션 추가
- 대시보드 KPI를 운영 중심 숫자로 재정렬
- 예약 관리 탭 구조를 실무형 상태로 변경
- 예약 관리 테이블에 결제 상태 / 정산 상태 / 이슈 상태 추가
- 일일 정산 상단 KPI를 정산용 숫자로 교체
- 일일 정산에 정산 원장 테이블 추가
- 현금 시재에 차액/사유/승인 구조 추가
- 예외/보류 영역 추가

## P1

- 지점별 현황 위젯 추가
- 예약 상세 슬라이드 패널 추가
- 대시보드 ↔ 예약 ↔ 정산 클릭 흐름 연결
- 지점별 마감 상태 카드 추가
- 리포트 다운로드 정리

## P2

- 월 정산 화면 연결
- 파트너/운송사 지급 예정 구조 추가
- 이슈/차액 승인 플로우 추가
- 정산서 출력 기능 연결

# 9. 안티그래비티 작업 요청 포맷

## 작업 목표

본사 어드민을 단순 조회형 페이지 집합이 아니라, **운영-예약-정산이 이어지는 실무형 관리 시스템**으로 개편한다.

## 작업 원칙

- 기존 브랜드/톤앤매너는 유지
- 과한 시각 효과보다 정보 우선순위와 실무 동선 개선에 집중
- 테이블/카드 중심 정보구조는 유지하되 역할을 명확히 분리
- 상태값은 운영/결제/정산으로 분리 표기
- 클릭 동선이 자연스럽게 이어지도록 설계

## 필수 산출물

1. 통합 관제 대시보드 개편 시안
2. 예약 관리 개편 시안
3. 일일 정산 개편 시안
4. 세 화면 간 연결 동선 설계
5. 상태 배지 체계 정리
6. 우선순위별 구현 제안

# 10. 최종 한 줄 정리

이번 개편의 핵심은 각 페이지를 따로 예쁘게 만드는 것이 아니라,

**대시보드는 상황판, 예약 관리는 작업판, 정산은 마감판**

이 역할을 분명히 나누고, 세 화면이 하나의 운영 흐름처럼 이어지게 만드는 것이다.
