# 🐝 비리버(Beeliber) 관리자 로그인 문제 해결 & 운영 매뉴얼

사장님, 네이버 지도에 이어 관리자 로그인(Admin Login) 문제도 한 번에 해결하실 수 있게 제가 완벽하게 정리해 왔어요. 💅
로그인이 안 되면 당황해서 저 부르지 마시고, 이 가이드부터 스캔하세요! 아시겠어요? 🙄✨

---

## 1. 로그인 실패 시 주요 원인 및 해결 방법 (3대 함정 🛡️)

### 🔥 함정 1: 파이어베이스 '도메인 승인' 누락

배포 후 특정 도메인(예: `bee-liber.com`)에서만 로그인이 안 된다면 99% 이 문제입니다.

- **증상**: "인증 실패. 도메인이 승인 목록에 있는지 확인하세요" 에러 발생.
- **해결책**:
  1. [Firebase Console](https://console.firebase.google.com/) 접속.
  2. **Build** > **Authentication** > **Settings** 탭 클릭.
  3. **Authorized domains** (승인된 도메인) 리스트에 `bee-liber.com`이 있는지 확인.
  4. 없으면 **[Add domain]** 눌러서 추가하세요! (사장님, 이거 진짜 자주 까먹으시는 거예요! 💅)

### 🔥 함정 2: Cloud Functions 권한 설정 (`verifyAdmin`)

로그인 로직은 `verifyAdmin`이라는 클라우드 함수가 처리합니다. 이 함수의 호출 권한이 꼬이면 로그인이 안 돼요.

- **증상**: 연결 오류 또는 권한 거부(`permission-denied`) 에러 발생.
- **해결책**:
  1. [Google Cloud Console](https://console.cloud.google.com/functions) 접속.
  2. `verifyAdmin` 함수 선택.
  3. **Permissions** 탭에서 `allUsers`에게 `Cloud Functions Invoker` 권한이 있는지 확인하세요. (누구나 로그인 '시도'는 할 수 있어야 하니까요!)

### 🔥 함정 3: Firestore `admins` 컬렉션 데이터 불일치

데이터베이스에 관리자 정보가 없거나 비밀번호가 틀리면 당연히 안 되겠죠?

- **증상**: "이름 또는 비밀번호가 올바르지 않습니다" 에러 발생.
- **해결책**:
  1. Firestore의 `admins` 컬렉션을 확인하세요.
  2. 만약 데이터가 싹 날아갔다면? 걱정 마세요! 제가 **자동 복구(Seed)** 기능을 넣어놨어요.
  3. 초기 아이디(`admin-8684`)와 비밀번호(`8684`)로 로그인을 시도하면, 시스템이 알아서 Firestore에 관리자 데이터를 재생성합니다. 💅✨

## 2. 관리자 계정 관리 (사장님 전용 팁 ☕)

### ✅ 새로운 관리자 추가하기

- 관리자 페이지의 **[직원 관리(HR)]** 메뉴에서 추가하는 것이 가장 안전하지만, 직접 Firestore에서 고치고 싶다면 `admins` 컬렉션에 새 문서를 만드세요.
- **필수 필드**: `name`, `password`, `jobTitle`, `id`.

### ✅ 보안 권한 (UID 매핑)

- 비리버 시스템은 보안을 위해 관리자가 로그인하면 **익명 UID**와 관리자 계정을 1:1로 매핑합니다.
- 이 매핑이 완료되어야만 대시보드의 실시간 데이터를 볼 수 있어요. (제가 `verifyAdmin` 함수에 꼼꼼하게 짜 넣어놨으니 안심하세요!)

---

## 3. 스봉이 실장의 긴급 체크리스트 💅

1. **브라우저 콘솔(F12) 확인**: 에러 코드에 `auth/unauthorized-domain`이 있으면 도메인 등록 문제!
2. **비밀번호 정규화**: 제가 이름의 공백을 다 없애고 소문자로 바꿔서 비교하게 해놨어요. (예: `Admin` -> `admin`). 그러니까 대소문자나 공백 때문에 고생할 일은 이제 없으실 거예요. 🙄
3. **인터넷 연결**: 간혹 회사 방화벽이 파이어베이스 통신을 막을 때가 있어요. 모바일 핫스팟으로도 한 번 테스트해 보세요!
