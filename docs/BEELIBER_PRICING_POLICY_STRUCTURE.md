## 4. 계산 로직

### 4-1. 기본 규칙
- `0 < t <= 4h` → 4시간 기본 요금 1회
- `4h < t < 24h` → 4시간 기본 요금 + 1시간 추가 요금 누적
- `t = 24h` → 24시간 첫날 요금 적용
- `24h < t` → 24시간 첫날 요금 + 추가 일자 보관비 누적

### 4-2. 수식 기준

#### 첫날 24시간 이내
- `first_day_price = price_4h`, if `0 < t <= 4`
- `first_day_price = price_4h + ceil(t - 4) * hourly_after_4h`, if `4 < t < 24`
- `first_day_price = price_day1`, if `t = 24`

#### 24시간 초과
- `total_price = price_day1 + ceil((t - 24) / 24) * extra_day_storage`

### 4-3. 시간 단가 산정 원칙
- 손가방: `(8,000 - 4,000) / 20 = 200원/시간`
- 캐리어: `(10,000 - 5,000) / 20 = 250원/시간`
- 유모차, 자전거: `(14,000 - 10,000) / 20 = 200원/시간`

### 4-4. 운영 해석
- 기준 시각은 결제 완료 시각 또는 입고 처리 시각입니다.
- 분 단위는 올림 처리합니다.
- 4시간 이후는 `1시간 단위`로 과금합니다.
- 24시간을 넘긴 뒤에는 보험료를 다시 붙이지 않고, 추가 일자 보관비만 누적합니다.

### 4-5. 계산 예시

#### 손가방
- 2시간 = 4,000원
- 5시간 = 4,200원
- 8시간 = 4,800원
- 12시간 = 5,600원
- 24시간 = 8,000원

#### 캐리어
- 3시간 = 5,000원
- 5시간 = 5,250원
- 8시간 = 6,000원
- 12시간 = 7,000원
- 24시간 = 10,000원

#### 유모차, 자전거

## 5. 프론트 노출 구조

### 5-1. 고객용 메인 가격 메시지
- 쇼핑백, 손가방: `4시간 4,000원부터 / 이후 1시간당 200원 / 1일 8,000원`
- 캐리어: `4시간 5,000원부터 / 이후 1시간당 250원 / 1일 10,000원`
- 유모차, 자전거: `4시간 10,000원부터 / 이후 1시간당 200원 / 1일 14,000원`

### 5-2. 보조 메시지 원칙
- 기본은 4시간 이용 요금
- 4시간 이후는 1시간 단위 추가
- 24시간을 넘기면 추가 일자 보관비만 누적
- 보험료는 첫날에만 1회 반영
- 저렴함보다 `간단함 / 안심 / 자유로운 이동`을 강조

### 5-3. 상세 페이지 안내 문구 구조
- 첫 4시간은 기본 요금으로 계산
- 4시간 이후는 1시간 단위로 추가 계산
- 하루를 넘기면 추가 일자 보관비가 붙음
- 보험료는 첫날에만 포함
- 지점별 보관 가능 품목은 다를 수 있음

- 물품 시세 50만원 초과 또는 고가품 판단 시 접수 차단

## 11. 현재 반영 상태와 다음 체크

1. 기존 데이터 중 배송 예약에 잘못 들어간 `유모차, 자전거(XL)` 건은 운영 판단 후 별도 정리합니다.

3. 고객용 가격 카피와 랜딩 SEO 문구가 현재 분류 체계와 완전히 일치하는지 계속 점검합니다.
